## Overview

Farm Analytics Dashboard is a lightweight data engineering and visualization project that demonstrates a complete ETL (Extract, Transform, Load) pipeline — from raw CSV ingestion to an interactive browser-based dashboard.

The project processes a synthetic agricultural dataset of 10,000 farm records across four crop types (Wheat, Corn, Rice, Soybean), stores the cleaned data in a structured SQLite database, and surfaces insights through an interactive Streamlit dashboard with dynamic filters and Plotly visualizations.

---

## Architecture

```
data/
└── farmer_advisor_dataset.csv          # raw source data
        │
        ▼
  process.py                            # ETL: cleaning + transformation
        │
        ▼
  cleaned.csv                           # cleaned intermediate output
        │
        ▼
  db.py                                 # data layer: loads into SQLite
        │
        ▼
  data.db                               # SQLite database (farm_data table)
        │
        ▼
  app.py                                # Streamlit dashboard (frontend)
```

Each stage is independently executable, making the pipeline easy to debug, extend, or replace at any layer.

## Getting Started

### Prerequisites

- Python 3.8+
- pip

### Installation

**1. Clone or download the project**

```bash
git clone https://github.com/your-username/farm-dashboard.git
cd farm-dashboard
```

**2. Create and activate a virtual environment** *(recommended)*

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

**3. Install dependencies**

```bash
pip install pandas streamlit plotly
```

> SQLite is built into Python — no installation required.

## Pipeline

Run the three scripts in order. Each step depends on the output of the previous one.

### Step 1 — Data Cleaning

```bash
python process.py
```

Reads the raw CSV, performs cleaning and transformation, and writes `cleaned.csv`.

**What it does:**
- Drops rows with missing values (precautionary — dataset is clean)
- Rounds all numeric columns to 2 decimal places for realism, except `Crop_Yield_ton` which retains full precision as the primary output metric
- Resets the DataFrame index after transformation
- Saves output to `cleaned.csv`

---

### Step 2 — Load into SQLite

```bash
python db.py
```

Reads `cleaned.csv` and loads it into a SQLite database (`data.db`), creating the `farm_data` table with `Farm_ID` as the primary key.

**What it does:**
- Connects to (or creates) `data.db`
- Drops and recreates the `farm_data` table to ensure a fresh load on each run
- Defines the table schema explicitly, including `Farm_ID INTEGER PRIMARY KEY`
- Inserts all rows from `cleaned.csv`
- Verifies the load with a row count query

---

### Step 3 — Launch Dashboard

```bash
streamlit run app.py
```

Starts the Streamlit server and opens the dashboard in your default browser at `http://localhost:8501`.

> **Windows note:** if the above command fails, use:
> ```bash
> python -m streamlit run app.py
> ```

## Dashboard Features

### KPI Cards
Four summary metrics displayed at the top of the dashboard, dynamically updated based on active filters:

| Metric | Description |
|--------|-------------|
| Avg Crop Yield | Mean yield in tons across filtered farms |
| Avg Sustainability Score | Mean sustainability score (0–100) |
| Avg Fertilizer Usage | Mean fertilizer applied in kg |
| Avg Pesticide Usage | Mean pesticide applied in kg |

---

### Sidebar Filters
All filters update every chart and KPI card in real time:

| Filter | Type | Description |
|--------|------|-------------|
| Crop Type | Dropdown | Filter by a specific crop or view all |
| Crop Yield (ton) | Range slider | Set min/max yield threshold |
| Sustainability Score | Range slider | Filter by sustainability range |
| Soil pH | Range slider | Narrow by soil acidity level |
| Rainfall (mm) | Range slider | Filter by rainfall range |

---

### Charts

**Crop Yield Analysis**
- Bar chart — average yield by crop type
- Pie/donut chart — crop type distribution across filtered farms

**Distributions**
- Histogram — crop yield distribution, overlaid by crop type
- Histogram — sustainability score distribution, overlaid by crop type

**Soil & Weather vs Yield**
- Scatter plot — soil pH vs crop yield, colored by crop type
- Scatter plot — rainfall vs crop yield, colored by crop type

**Fertilizer & Pesticide Usage**
- Bar chart — average fertilizer usage by crop type
- Scatter plot — fertilizer usage vs crop yield

**Sustainability Score Trends**
- Bar chart — average sustainability score by crop type
- Scatter plot — pesticide usage vs sustainability score

**Raw Data Table**
- Collapsible table showing all filtered rows with row count

## Dataset

**Source:** `farmer_advisor_dataset.csv`  
**Records:** 10,000 farms  
**Columns:** 10

| Column | Type | Description |
|--------|------|-------------|
| Farm_ID | Integer | Unique farm identifier (primary key) |
| Soil_pH | Float | Soil acidity level (5.5 – 7.5) |
| Soil_Moisture | Float | Soil moisture percentage (10 – 50) |
| Temperature_C | Float | Ambient temperature in Celsius |
| Rainfall_mm | Float | Annual rainfall in millimeters |
| Crop_Type | String | Crop variety: Wheat, Corn, Rice, Soybean |
| Fertilizer_Usage_kg | Float | Fertilizer applied in kilograms |
| Pesticide_Usage_kg | Float | Pesticide applied in kilograms |
| Crop_Yield_ton | Float | Crop yield in tons (primary output metric) |
| Sustainability_Score | Float | Composite sustainability score (0 – 100) |

**Data quality:** No missing values. All numeric ranges are realistic. `Crop_Type` contains exactly 4 balanced categories.

---

## Known Limitations

- **No time dimension** — the dataset does not include a year or date column, so time-series analysis is not possible
- **Synthetic data** — generated programmatically; relationships between variables (e.g., rainfall and yield) may not reflect real agricultural patterns
- **Single-table schema** — the database uses one flat table; a production system would normalize across multiple relational tables
- **No authentication** — the Streamlit dashboard has no login layer; not suitable for deployment without access controls
- **Static dataset** — the pipeline is batch-oriented; there is no live data ingestion or scheduled refresh mechanism
